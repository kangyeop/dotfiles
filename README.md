# dotfiles

Personal Claude Code, Codex, and zsh configuration, synced across machines.

## Prerequisites

Install these yourself before running `install.sh` — this repo only symlinks config, it doesn't install tooling:

- [Homebrew](https://brew.sh)
- [oh-my-zsh](https://ohmyz.sh) + [powerlevel10k](https://github.com/romkatv/powerlevel10k)
- [nvm](https://github.com/nvm-sh/nvm), rbenv, [bun](https://bun.sh)
- `fzf`, `zsh-syntax-highlighting`, `zsh-autosuggestions` (zsh plugins referenced in `.zshrc`)
- [Claude Code](https://claude.com/claude-code) CLI, plus the `oh-my-claude-sisyphus` package (manages `~/.claude/skills` and `~/.agents/skills`)
- Codex CLI, plus `oh-my-codex` (manages `~/.codex/prompts`)

## Usage

```sh
git clone <this-repo-url> ~/dotfiles
cd ~/dotfiles
./install.sh
```

`install.sh` is idempotent. If a target file already exists (and isn't already the right symlink), it's moved to `~/.dotfiles-backup/<timestamp>/` before the symlink is created.

## What's tracked

| Repo file | Symlinked to |
|---|---|
| `zsh/zshrc` | `~/.zshrc` |
| `zsh/p10k.zsh` | `~/.p10k.zsh` |
| `claude/settings.json` | `~/.claude/settings.json` |
| `claude/mcp.json` | `~/.claude/.mcp.json` |
| `claude/statusline-command.sh` | `~/.claude/statusline-command.sh` |
| `codex/config.toml` | `~/.codex/config.toml` |

## What's deliberately NOT tracked

- `~/.claude/skills/`, `~/.agents/skills/` — managed by `oh-my-claude-sisyphus` via its own lockfile. Reinstalling that tool regenerates them.
- `~/.codex/prompts/` — managed by `oh-my-codex`; regenerated by its own setup.
- Claude Code's plugin cache (`~/.claude/plugins/`) — plugins are declared in `settings.json`'s `enabledPlugins`/`extraKnownMarketplaces` (which *is* tracked), and Claude Code reinstalls them from that on demand.
- Any session/history/cache/sqlite state — machine-local, never portable.

## Known caveats (hand-fix per machine)

- `claude/mcp.json`'s `filesystem` MCP server hardcodes `/Users/yeop/dev` as an arg. JSON has no shell-variable expansion — if the username or dev-folder path differs on a machine, edit this by hand after cloning.
- `codex/config.toml`'s `notify` hook embeds an absolute path through a specific nvm Node version. This is regenerated by `oh-my-codex`'s own setup — if it's stale, re-run that tool's setup rather than hand-editing.

## Secrets

None are stored here. The `tavily` MCP server (which required an API key) was intentionally dropped from `mcp.json`. If you add an MCP server that needs a secret, keep the secret out of this repo (env var, local-only override) rather than committing it.
