# Dotfiles Sync — Design

## Purpose

Sync Claude Code, Codex, and zsh configuration across multiple machines (macOS + Linux) via a private GitHub repo, so setting up a new machine is a `git clone` + `./install.sh` away.

## Scope

**In scope:**
- Claude Code: `~/.claude/settings.json`, `~/.claude/.mcp.json`, `~/.claude/statusline-command.sh`
- Codex: `~/.codex/config.toml`
- zsh: `~/.zshrc`, `~/.p10k.zsh`

**Out of scope (explicitly not tracked):**
- `~/.claude/skills/`, `~/.agents/skills/` — managed by the `oh-my-claude-sisyphus` package via its own lockfile (`~/.agents/.skill-lock.json`); reinstalling the tool regenerates these.
- `~/.codex/prompts/` — managed by `oh-my-codex`; regenerated by that tool's own setup.
- Claude Code plugin cache (`~/.claude/plugins/`) — plugins are declared in `settings.json.enabledPlugins`/`extraKnownMarketplaces` (which we do track) and Claude Code reinstalls them from there.
- Any session/history/cache/sqlite state (machine-local, never portable).
- Prerequisite tool installation (Homebrew, oh-my-zsh, powerlevel10k, nvm, rbenv, bun, oh-my-codex, oh-my-claude-sisyphus) — documented in README as manual prerequisites, not automated.

## Repo layout

```
~/dotfiles/
├── install.sh
├── README.md
├── zsh/
│   ├── zshrc
│   └── p10k.zsh
├── claude/
│   ├── settings.json
│   ├── mcp.json
│   └── statusline-command.sh
└── codex/
    └── config.toml
```

File mapping (repo path → symlink target):

| Repo file | Symlink target |
|---|---|
| `zsh/zshrc` | `~/.zshrc` |
| `zsh/p10k.zsh` | `~/.p10k.zsh` |
| `claude/settings.json` | `~/.claude/settings.json` |
| `claude/mcp.json` | `~/.claude/.mcp.json` |
| `claude/statusline-command.sh` | `~/.claude/statusline-command.sh` |
| `codex/config.toml` | `~/.codex/config.toml` |

The repo holds the real file content. The home-directory path becomes a symlink pointing into the repo, so future edits at the live location are automatically tracked by git.

## Sanitization & portability

- **`claude/mcp.json`**: the `tavily` MCP server entry (which embedded a live `TAVILY_API_KEY`) is removed entirely. Only `context7` and `filesystem` remain.
- **`claude/settings.json`**: the statusline command is rewritten from the hardcoded `bash /Users/yeop/.claude/statusline-command.sh` to `bash "$HOME/.claude/statusline-command.sh"`, so it works under any username/OS.
- **Known caveats (documented in README, not auto-fixed):**
  - `claude/mcp.json`'s `filesystem` server hardcodes `/Users/yeop/dev` as an arg. JSON has no shell-variable expansion, so this must be hand-edited per machine if the username or dev-folder path differs.
  - `codex/config.toml`'s `notify` hook embeds an absolute path through a specific nvm Node version (e.g. `v22.9.0`). This is regenerated by `oh-my-codex`'s own setup tooling and isn't something `install.sh` should try to template.

## install.sh behavior

- Pure POSIX-compatible bash (works on macOS and Linux).
- Iterates a fixed list of (repo path, target path) pairs.
- For each pair:
  - If the target is already a symlink pointing at the repo file, skip.
  - If the target exists as a real file/dir, move it to `~/.dotfiles-backup/<timestamp>/` before linking.
  - Otherwise, `mkdir -p` the parent directory and create the symlink.
- Idempotent — safe to re-run.
- No attempt to install prerequisites (Homebrew, oh-my-zsh, nvm, oh-my-codex, etc.) — that's manual, per README.

## README.md contents

- One-line description of what this repo is.
- Prerequisites checklist (tools that must already be installed before running `install.sh`): Homebrew, oh-my-zsh + powerlevel10k, nvm, rbenv, bun, oh-my-codex, oh-my-claude-sisyphus.
- Usage: `git clone <repo-url> ~/dotfiles && cd ~/dotfiles && ./install.sh`.
- Explicit note that skills/prompts are intentionally not tracked here (owned by the tools above).
- Explicit note of the two hardcoded-path caveats above and how to fix them by hand on a new machine.

## Testing

- Manual: run `install.sh` on this machine (files already exist for real) and confirm backups land in `~/.dotfiles-backup/` and symlinks resolve correctly.
- Manual: verify `install.sh` is idempotent by running it twice.
- Manual: `zsh -n zsh/zshrc` and equivalent syntax checks where applicable; validate JSON files parse (`jq .`) and TOML parses.
